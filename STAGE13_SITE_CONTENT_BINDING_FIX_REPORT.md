# Исправление связи админки с основным сайтом

## Что было сломано

1. Главная страница частично читала данные из статичных файлов, а не из сохранённых настроек админки.
2. Страница «О тренере» брала достижения и сертификаты из `defaultBrandProfile`, поэтому изменения из `/admin/brand` не отображались.
3. Фото бренда обновлялись только после перезагрузки отдельных компонентов и не слушали событие сохранения.
4. Instagram на сайте выводился через удаление `@`, поэтому казалось, что символ «собачка» исчезает.
5. Сохранение данных бренда не было централизовано.

## Что исправлено

1. Добавлен единый хук бренда:

`src/lib/brand/useBrandProfile.ts`

Он читает, сохраняет и обновляет данные личного бренда через одно хранилище.

2. Исправлен компонент фото:

`src/components/brand/BrandImage.tsx`

Теперь фото сразу реагируют на изменения из админки.

3. Исправлена страница «О тренере»:

`src/app/about/page.tsx`

Теперь достижения, сертификаты и роль тренера берутся из сохранённых данных бренда.

4. Исправлен блок «О тренере» на главной странице:

`src/components/sections/About.tsx`

Теперь достижения берутся из админки.

5. Исправлен Instagram:

`src/lib/site-content/useSiteContent.ts`
`src/components/ui/Footer.tsx`

Теперь в админке можно писать:

`@agashka.powerlifting`

и символ `@` сохраняется. Для ссылки он корректно убирается только внутри URL, а на сайте отображается с `@`.

6. Исправлена нормализация настроек сайта:

`src/lib/site-content/useSiteContent.ts`

Теперь тарифы, контакты, WhatsApp, Instagram, YouTube, стаж и тексты сохраняются стабильнее.

## Как проверять

1. Запустить проект.
2. Открыть:

`http://localhost:3000/admin/site-settings`

3. Изменить телефон, Instagram, YouTube, тарифы, заголовки.
4. Нажать «Сохранить».
5. Открыть главную:

`http://localhost:3000`

6. Обновить страницу `Ctrl + F5`.
7. Проверить блоки Hero, тарифы, контакты, Instagram.
8. Открыть:

`http://localhost:3000/admin/brand`

9. Изменить достижения, сертификаты, роль, фото.
10. Открыть:

`http://localhost:3000/about`

## Важно

В этой локальной версии данные сохраняются в браузере через `localStorage`.
Если открыть сайт в другом браузере, другом компьютере или после очистки данных браузера, данные могут вернуться к базовым.

Для настоящего боевого сайта на домене нужно подключить Supabase-таблицу `site_settings` и `trainer_brand`, чтобы данные хранились на сервере.

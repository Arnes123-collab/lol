# Stage 14 Save Fix Report

Исправлена ошибка, из-за которой изменения в `/admin/site-settings` и `/admin/brand` могли исчезать даже в самой админке.

## Причина ошибки

Админка сначала сохраняла данные в `localStorage`, но затем клиентский хук делал запрос к серверному API. Если Supabase ещё не подключён, API возвращал базовые значения (`default`). Эти базовые значения перезаписывали локально изменённые данные.

Из-за этого пользователь видел эффект:

1. Изменил тарифы, телефон, Instagram, YouTube или биографию.
2. Нажал сохранить.
3. После обновления страницы данные снова старые.

## Что исправлено

- `src/lib/site-content/useSiteContent.ts`
  - серверные базовые данные больше не перезаписывают локальные изменения;
  - сохранение работает локально даже без Supabase;
  - при подключённом Supabase данные будут сохраняться на сервер;
  - Instagram сохраняет символ `@`;
  - YouTube не очищается автоматически.

- `src/components/admin/site-settings/SiteSettingsManager.tsx`
  - админка теперь корректно показывает, где сохранены данные: локально или на сервере.

- `src/lib/brand/useBrandProfile.ts`
  - бренд тренера больше не сбрасывается к базовым значениям, если Supabase ещё не подключён;
  - фото, достижения, сертификаты и тексты бренда сохраняются локально.

- `src/components/admin/brand/BrandManager.tsx`
  - исправлены сообщения о сохранении;
  - локальное сохранение больше не считается критической ошибкой.

## Как проверить

1. Запустить проект.
2. Открыть `/admin/site-settings`.
3. Изменить тариф, телефон, Instagram с `@`, YouTube или биографию.
4. Нажать `Сохранить`.
5. Обновить страницу админки через F5.
6. Данные должны остаться.
7. Открыть главную `/` и нажать Ctrl+F5.
8. Изменения должны отображаться на сайте.

## Важно

В локальной версии данные сохраняются в браузере. На домене для общего сохранения на всех устройствах нужно подключить Supabase и выполнить SQL из Stage14.

# AGASHKA POWERLIFTING - Этап 9

## Что сделано

### 1. Безопасность админки

- Добавлен `middleware.ts`.
- Все страницы `/admin/*`, кроме `/admin/login`, закрыты cookie-сессией.
- Защищены админские API:
  - `/api/training-plans/*`
  - `/api/training-plan-items/*`
  - `/api/exercises/*`
- Добавлен выход из админки: `/admin/logout`.
- Демо-пароль `admin123` теперь работает только локально.
- В production без `ADMIN_PASSWORD` вход блокируется.

### 2. Финальная страница проверки запуска

Добавлена страница:

`/admin/launch-checklist`

Она проверяет:

- задан ли `ADMIN_PASSWORD`
- заданы ли переменные Supabase
- доступность таблиц:
  - `clients`
  - `training_plans`
  - `client_payments`
  - `training_reports`
- готовность мобильной версии
- наличие процедуры резервного копирования

### 3. Supabase

- Обновлён `src/lib/supabase.ts`.
- Если Supabase ещё не настроен, проект не пытается обращаться к фейковому адресу и не падает при локальной проверке.
- Добавлен SQL-файл:

`sql/stage9_launch_notes.sql`

### 4. Резервное копирование

Добавлен скрипт:

`scripts/backup-supabase.sh`

Он делает SQL-бэкап Supabase через `pg_dump`, если задан `DATABASE_URL`.

Добавлена инструкция:

`docs/LAUNCH_CHECKLIST.md`

### 5. Мобильная версия

- Админка получила ссылку на финальную проверку запуска.
- Кнопки в админке адаптированы в несколько строк.
- Основные разделы уже используют мобильные сетки, горизонтальный скролл таблиц и адаптивные отступы.

### 6. Исправление PDF-сборки

Проблема: `jspdf` в серверной сборке Next.js требовал дополнительные зависимости и ломал production build.

Исправление:

- `PlanPdfButton` переведён на клиентскую загрузку PDF-библиотек через CDN.
- Если PDF-библиотеки не загрузились, автоматически открывается печать, чтобы клиент мог сохранить план как PDF через браузер.

```bash
npx tsc --noEmit
```

Результат: TypeScript без ошибок.

`npm run build` в песочнице начал production-сборку, но процесс Next.js завис/не завершился в лимит времени окружения. Основная компиляция проходила дальше после исправления `jspdf`, но полную production-проверку нужно повторить локально или на Vercel.

## Что нужно сделать на Vercel

В Environment Variables добавить:

```text
ADMIN_PASSWORD=ваш_сильный_пароль
NEXT_PUBLIC_SUPABASE_URL=ваш_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш_supabase_anon_key
```

После деплоя открыть:

`/admin/launch-checklist`

## Важное предупреждение

Сейчас это хороший MVP для тренера: заявки, планы, PDF, оплаты, отчёты и админка.

Но это ещё не банковский уровень безопасности. Не храните паспортные данные, банковские карты и лишние медицинские диагнозы. Для следующего уровня нужно перевести админку на Supabase Auth или server-only service role.
